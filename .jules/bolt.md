## 2024-05-24 - [Avoid Array.find() inside Array.reduce() when map exists]
**Learning:** In `app/api/admin/route.js`, the code was performing an O(N*M) calculation to get total loaned items by iterating over all items and finding the corresponding loan for each item to check its status. However, a Map of items by loan was already created immediately before this calculation, and the items were already assigned to each loan (`loan.items`).
**Action:** When calculating aggregates across related entities (like loans and their items), always check if a relational map or grouped structure has already been built in the preceding lines. Iterating over the pre-grouped structure (e.g., `formattedLoans.filter(...).reduce(...)`) turns an O(N*M) operation into an O(N) operation and is often cleaner.
